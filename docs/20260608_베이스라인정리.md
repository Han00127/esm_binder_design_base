## 1. 배경 — 왜 이 baseline인가

- 기존 `esmcraft`(distogram-gradient CDR hallucination) 실험에서 confidence-loss-in-gradient,
  MSA injection 등을 탐색했으나 명확한 이득을 확인 못함 → **잠시 보류**.
- 의사결정: **"논문에서 제시한 방법(Algorithm 11 + 후처리 랭킹)을 *그대로* 재현해 foundry baseline으로
  고정"** 하고, 그 위에 개선을 얹는다. (재현 가능한 기준선이 있어야 개선의 효과를 격리 측정 가능.)
- base 모델 선정 근거: 별도 벤치마크(antibody-antigen 구조예측, TM/DockQ/CA-RMSD)에서
  **base ESMFold2 > ESMFold2-Experimental** 확인 (Experimental은 over-confident: ipSAE 높지만 구조 틀림).
  → 생성·critic 양쪽에서 base 계열 사용.

## 2. 방법론 — Algorithm 11 (Gradient-Guided Binder Sequence Optimization)

### 2.1 입력/표현
- 복합체 = **[항원(target, 고정) | scFv(binder)]**, scFv = VH-linker(GGGGS×3)-VL.
- 설계 변수 `x ∈ R^{L×20}` (위치별 20 AA logits). softmax(x/T)로 soft 서열 분포.
- **고정(framework·항원):** init logit=10 → softmax ~one-hot(native AA), gradient mask=0 → 불변.
  단 **forward에는 항상 들어가 구조 문맥(Ig fold scaffold)을 제공.** ("고정=업데이트 X"이지 "입력 제외"가 아님.)
- **설계(CDR mutable):** init ~N(0,1e-4) (≈균등, 미세 noise로 대칭깸), Cys logit=-1e6(시스테인 배제),
  gradient mask=1 → 업데이트 대상.

### 2.2 최적화 루프 (k=1..K=150)
1. **온도 anneal:** `T_k = T_min + (1-T_min)·½(1+cos(πk/K))`, T_min=0.01 (1→0.01).
2. **LR:** `α_k = α_max·T_k`, α_max=0.1.
3. **soft = softmax(x/T_k).**
4. **구조 forward:** soft → res_type[1,L,33] → ESMFold2 trunk → **distogram_logits[1,L,L,64]** (미분가능,
   2-22Å·64bin). diffusion sampler 최소(num_sampling_steps=1) = 논문 "trunk and distogram only" 근사.
5. **구조 손실:** `L_struct = λ_intra·IntraContact + λ_inter·InterContact + λ_glob·Globularity`
   (λ_intra=λ_inter=0.5, λ_glob=0.2).
   - InterContact: CDR↔epitope 접촉 (-log P(dist<20Å), topk).
   - IntraContact: scFv 내부 접촉 (cutoff 14Å, seqsep≥9).
   - Globularity: scFv 평균 기대거리(컴팩트성), DISTOGRAM_MAX로 정규화.
6. **LM prior(자연성):** ESMC-600M masked pseudo-PPL (M=4 passes, r_mask=0.15) → `L_LM`.
   - soft 문맥(option a, 충실): soft 임베딩을 ESMC에 주입, CDR 일부를 mask해 자연성 평가.
7. **성분별 gradient 정규화:** `ĝ = √n_mut·(g⊙mask)/||g⊙mask||`. 구조·LM 각각 정규화.
8. **결합·SGD:** `g = ĝ_struct + λ_LM·ĝ_LM` (λ_LM=0.05 antibody), `x ← x − α_k·g`.
9. **b\* 추적 (저온 T<0.05):** ↓ 2.3 참조.

### 2.3 b\* 선택 — 충실 vs 근사 (★ 핵심 구현 포인트)
논문 line 10-15:
- **T_k ≥ 0.05:** distogram-only forward (손실용 D_k).
- **T_k < 0.05:** `(D_k, ipTM_k) = F(soft; loops=1, steps=50, confidence head)` → **진짜 ipTM**로
  `if ipTM_k > ipTM*: ipTM*=ipTM_k; b*=argmax(soft_binder)`.

**충실버전 구현(현재 기본):**
- 저온 step에서 **현재 설계 argmax → 이산 scFv → real confidence head로 fold(no_grad, steps=50)**
  → 항원↔scFv **진짜 ipTM** → b\* 추적.
- confidence head는 평소 NoOp(distogram-only gradient, 저렴)이고, 저온에서만 **real head를 swap-in**해
  no_grad fold 후 swap-back. **gradient는 distogram만 통과(backward 안전).**
- T<0.05에서 softmax≈one-hot이라 argmax≈soft (논문 충실). 또한 논문 b\*=argmax(soft)이므로
  **실제 반환할 설계를 그대로 평가** = 오히려 더 정합적.
- `--no-real-iptm` 토글 시 **distogram-ipTM proxy**로 폴백(근사버전, A/B용).

### 2.4 후처리 랭킹 (논문 후처리)
- 생성된 후보들 → `disto_iptm`(이제 real ipTM) 상위 top-K 선별.
- 각 후보 × **4 Experimental critic** (full_2021, full_2025, fast_2021, fast_2025)로 폴딩(MSA=auto)
  → `metrics.evaluate_complex`로 인터페이스 **ipSAE** → **critic 평균 → 랭킹**.
- critic은 forward-only RANKING 전용 (구조 아키텍처가 base와 달라 gradient 생성엔 부적합).


## 4. 논문과의 차이점 (architectural)

| 항목 | 논문 | 현재 구현 | 정합성 |
|---|---|---|---|
| 구조 손실용 distogram D_k | trunk distogram | 동일 (trunk distogram) | ✅ 충실 |
| 저온 ipTM_k | confidence head(steps=50) | real confidence fold(no_grad, argmax 입력) | ✅ 충실(argmax≈soft @저온) |
| LM prior | ESMC MaskedPseudoPPL | ESMC-600M, soft 문맥 | ✅ 충실 |
| gradient 흐름 | distogram 경로 | distogram만 (confidence backward X) | ✅ (저온 ipTM은 forward-only) |
| 고온 distogram sampler | trunk+distogram only | num_sampling_steps=1 근사 | ◯ 근사(의도 동일) |
| 후처리 critic 수 | 4(low-compute)/19(high) | 4-critic | ◯ low-compute 모드 |

**메모리/속도(검증):** 생성 ~11.4s/step (6B trunk-bound), 저온 fold +~3s/step (저온 ~20 step만),
풀 K=150 = ~56분 생성 / ~12분 랭킹, GPU당 58GB(80GB 내), OOM 없음.
**OOM 회피 기법:** 구조·LM gradient를 별도 softmax 두 그래프로 분리(공존 X).

(예: `TSLNQWFYTYYWWSPP...WYAYQ`). → **baseline은 약하나 명확한 출발점.** (↓ 9장 원인)

### 6.2 충실버전 (real ipTM b*, λ_LM=0.05) — `runs/`
풀 K=150, 12후보 → top6 4-critic. 유일한 차이 = **저온 b\* 선택(proxy→real ipTM)**.

| 순위 | 후보 | avg ipSAE | in-loop b*(real ipTM) | full21 | full25 | fast21 | fast25 |
|---|---|---|---|---|---|---|---|
| 1 | s1 | **0.122** | 0.355 | **0.475** | 0.00 | 0.00 | 0.01 |
| 2 | s400 | 0.014 | 0.319 | 0.058 | 0 | 0 | 0 |
| 3-6 | s100/200/101/0 | ≤0.010 | 0.30~0.46 | ~0 | ~0 | ~0 | ~0 |

**Baseline 대비:** top avg_ipsae 0.145(proxy) vs **0.122(충실)** — **개선 없음**(둘 다 단일-critic 아티팩트, critic 합의 0).

**핵심 발견:**
1. 충실 b\* 정확 구현 ✅ — in-loop 신호 proxy 0.07 → **real ipTM 0.30~0.46**(좋은 step 선택).
2. 그런데 **최종 4-critic avg_ipsae는 개선 없음** → **병목 = b\* 선택이 아니라 설계 현실성**(예측대로).
3. **★ 설계모델 자신의 ipTM이 독립 critic으로 전이 안 됨** — real ipTM 0.3~0.46인 설계가 critic에선 ~0.
   = 최적화가 distogram뿐 아니라 **confidence head까지 적대적으로 속임**(over-confidence, 벤치마크의
   Experimental over-confidence와 동일 현상).

**결론:** 충실버전 = 올바른 논문 재현 baseline(foundry 확정). 단 **논문 b\* 메커니즘만으로는 전이 가능한
binder 안 나옴** → 진짜 레버는 **설계 현실성**(7장 aromatic 해결: composition-KL + loss-rebalance +
항체 LM prior). 8장 로드맵의 Tier2/Tier1이 최우선.

## 7. 알려진 문제 — aromatic 병리 (W/Y/F 도배)

### 7.1 원인 (gradient hallucination의 근본 실패 모드)
1. **적대적 최적화:** "진짜 binder"가 아니라 "ESMFold2가 binder로 *점수 매기는* 입력"을 찾음 →
   adversarial example.
2. **모델의 학습된 지름길 "aromatic=접촉":** W/Y/F는 실제 코어·인터페이스에 많음 → 모델이 상관 학습 →
   접촉 손실 gradient가 **aromatic 방향**을 가리킴. (손실 자체가 끌어당김.)
3. **블롭이 구조손실을 공짜로 만족:** aromatic 덩어리는 빽빽 packing → 접촉↑·globularity↓ → 모델 confident.
4. **유일한 반대힘(LM)이 5%:** 성분별 정규화 후 `g=ĝ_struct+0.05·ĝ_LM` → LM이 구조의 1/20 → 압도당함.
5. **조성/개발성 제약 없음:** 바닐라 Alg11엔 LM prior가 유일 안전장치.
6. **저온 anneal이 못박음:** 고온에서 aromatic이 logit 질량 선점 → one-hot commit.

### 7.2 올바른 목표 (중요)
**aromatic 제거 ≠ 목표.** 실제 항체 paratope는 원래 **Tyr-rich**(Tyr가 가장 enriched, π-stacking/cation-π로
결합 핵심). 목표는 **자연 CDR 분포(방향족 ~15-25%)로 되돌리는 것**. 현재 ~50-70% = degenerate corner.
→ 통일 원리: **"설계를 자연 항체 CDR manifold 안에 머물게 하라."** (좋은 aromatic 보존, 블롭 제거.)

## 8. 개선 로드맵 (브랜치 전략)

> baseline(이 레포 main) = foundry 고정. 아래는 **각자 브랜치**로 실험 후 A/B로 검증.

### Tier 1 — 올바른 prior (천장 최고)
- **`feat/antibody-lm-prior`**: ESMC(일반 단백질 LM) → **항체 특이 LM**(OAS 학습: AbLang2/IgLM/AntiBERTy/
  ESM-OAS)로 교체. 진짜 CDR 분포 인코딩(Tyr-rich지만 W/F-블롭 아님).
  - 제약: compute 노드 firewall → **가중치 Colab→Drive 경로로 확보** 필요.

### Tier 2 — 즉시 가능 (다운로드 0, 미분가능)
- **`feat/composition-kl`**: 자연 CDR AA 빈도 타깃 **KL 페널티** → 자연 수준 방향족 허용, 초과만 패널티 (surgical).
- **`feat/loss-rebalance`**: λ_glob·λ_intra ↓, λ_inter 상대 강화 → "블롭 인센티브" 제거(인터페이스 중심).
- **`feat/aromatic-softcap`**: `relu(f_WYF−0.25)²` 초과분만 패널티 (단순 손잡이).

### Tier 3 — 탐색/refinement
- **`feat/variable-cdr-length`**: **H3 길이 가변** — trajectory마다 길이 샘플링(예: 9~18) → scFv 재구성·
  features 재생성 → 고정길이 최적화. 길이 다양성↑(특히 H3). *한 최적화 안에서는 불가(고정크기 logits)*,
  trajectory별 샘플링이 표준(BindCraft 방식).
- **`feat/native-warmstart`**: init를 native CDR에서 (affinity-maturation형; de novo 의도와 trade-off).
- **`feat/developability-rank`**: 후처리에 노출 소수성/방향족 패치 패널티(응집성 필터).
- **`feat/faithful-iptm-soft`**: (선택) 저온 ipTM을 argmax 대신 soft 입력으로 (논문 글자 그대로).

**측정 타깃:** 방향족 분율 ~50-70% → **자연 ~15-25%**, Tyr 보존·W/F 과잉 감소, avg_ipsae·critic 합의 ↑.

**추천 순서:** Tier2(composition-kl + loss-rebalance + aromatic-softcap) 한 실험 → A/B →
효과 확인 후 Tier1(항체 LM)로 천장 올리기. variable-cdr-length는 병렬 브랜치로.

## 13. 개선 아이디어 상세 (2026-06-08 논의 기록)

8장 로드맵의 각 항목 + 추가 아이디어를 구현 관점에서 상세화. (브랜치 작업 전 참고용.)

### 13.1 자연 CDR 분포 타깃 (`feat/composition-kl`)
**아이디어:** 설계 CDR의 AA 조성이 *자연 항체 CDR 분포*에서 벗어나면 페널티.
- 손실: `L_comp = Σ_pos KL(p_pos ‖ q_pos)` — p=위치별 설계분포(soft 평균), q=위치별 자연 PSSM.
- **자연 수준 Tyr/방향족은 허용, 초과만 페널티** → 외과적(좋은 aromatic 보존, 블롭 제거). 비용 0(추가 forward 없음).

**데이터 (많을수록·계층화가 핵심):**
| DB | 규모 | 용도 |
|---|---|---|
| **OAS**(Observed Antibody Space) | unpaired ~20억+, paired ~150만+ | 주력. IMGT 넘버링·chain·CDR 경계 주석 |
| **SAbDab/Thera-SAbDab** | 수천(구조) | 고품질 소량, therapeutic 셋 |

**계층화 필수:** chain(H/L)별 · CDR(H1/H2/H3/L1/L2/L3)별 · **길이별**(특히 H3) · 종/germline별(human).

**위치별 PSSM 구축:**
1. 같은 CDR·같은 길이 서열을 **ANARCI(IMGT) 넘버링**으로 정렬.
2. (chain,CDR,길이,position) → 20-AA 빈도행렬 = PSSM. **pseudocount(Dirichlet)** 추가.
3. → 위치 의미 인코딩(H3 apex=Tyr/Gly-rich, anchor=D/A/M …).

> **PSSM = 항체 LM(13.4)이 암묵적으로 하는 일의 명시적·context-free 버전.** PSSM(싸고 즉시)→LM(context-aware)로 업그레이드.
> 제약: OAS bulk 다운로드 = firewall → Colab→Drive. ANARCI는 로컬 설치.

### 13.2 region-aware loss + construct 일반화 (`feat/loss-rebalance`)
- **현재** `L_struct=0.5·intra+0.5·inter+0.2·glob`. **intra·glob = 자기 packing 보상 → 블롭/방향족 유도.** inter=결합(원하는 것).
- **재균형:** λ_intra·λ_glob↓, λ_inter 유지·↑. (0 아님 — CDR 최소 folding 필요.)
- **construct 일반화(scFv/Fv/Fab/full):** glob·intra를 **전체가 아니라 가변도메인(Fv)·CDR 인근에만** 계산.
  - full IgG에서 전체 globularity는 CDR 영향 미미 → 무의미. `fold_idx`를 가변도메인으로 좁히면 region-aware.
  - Fv(2-chain)는 linker 없음·VH-VL 인터페이스 유지·인덱스 재계산. **설계는 Fv/Fab 권장(full 불필요·느림), full엔 graft.**

### 13.3 aromatic softcap (`feat/aromatic-softcap`, 백업)
- `L_arom = relu(f_arom − τ)²`, f_arom=mean(p_W+p_Y+p_F over CDR), τ≈0.25. τ 이하 자유.
- composition-KL의 blunt 버전(방향족만). **13.1/13.2 부족 시** 가드로.

### 13.4 항체 LM prior (`feat/antibody-lm-prior`) ★ 천장 최고
- **ESMFold2를 *대체하지 않음*. ESMC 슬롯만 교체하는 별개 gradient 가지.**
```
        soft x ─┬─ [구조: ESMFold2] soft→distogram→L_struct→g_struct
                └─ [서열: 항체 LM]  soft(CDR)→masked pseudo-PPL→L_LM→g_LM   ← ESMC 교체
        g = ĝ_struct + λ_LM·ĝ_LM  (매 step)
```
- ESMFold2=구조(불변), 항체 LM=서열 자연성. **매 step** soft에 적용(마지막 단계 아님). distogram은 100% ESMFold2.
- 모델: **AbLang2 / AntiBERTy / IgLM**(OAS 학습). 진짜 항체 CDR 분포 인코딩(Tyr-rich·블롭 아님·framework 호환).
- 비용 작음(수백MB). 제약: 가중치 다운로드(Colab→Drive).

### 13.5 가변 CDR 길이 (`feat/variable-cdr-length`)
- **한 trajectory 안 길이고정(고정크기 logits) → trajectory마다 H3 길이 샘플링**(예 9~18, 자연 분포).
  per-trajectory scFv 재구성+features 재생성 → 고정길이 최적화.
- **이득:** H3 길이=항체 다양성 주결정자 · epitope 기하 상보성(오목→긴 H3, 평평→짧은) · local minima 탈출 · 후단 선택 다양성 · 생물 현실성. **부수효과:** 맞는 길이면 *더 적은 방향족*으로 인터페이스 가능 → aromatic 압력 일부 해소(추정).

### 13.6 ★ per-CDR distogram 가중치 (신규 아이디어, 2026-06-08)
**아이디어:** CDR별 inter/intra 가중치 차등 — H3/L3=1.0, H2/L2=0.5, H1/L1=0.3.
- **타당:** paratope 기여 H3≫L3>H2/L2>H1/L1. 설계압력 H3 집중 = 올바른 bias. 주변 CDR 방향족 과적 완화.
- **한계(확인됨):** **H3(가중치1)는 여전히 방향족 블롭** — 가장 중요한 loop의 병목은 못 고침. 증상 위치만 축소.
- **주의:** (1) 과도히 낮추면 H1/H2가 epitope 무시(과소설계). (2) intra·glob=자기packing이라 **inter 가중치만으론 부족** → glob/intra 축소와 병행. (3) 더 강한 변형=mutable 집합 축소(H3/L3만 설계, 나머지 native 고정).
- **결론:** **loss-rebalance(13.2)의 일부로 채택**(주변 CDR). **H3 방향족은 13.1/13.4(identity-aware)** 로 — H3를 자연 H3 분포로 끌어 H3에서도 방향족을 자연 수준으로. → **per-CDR 가중치 + composition/LM 조합**이 정답에 근접.
